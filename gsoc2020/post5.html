August 10th, 2019

In the past week I implemented end-to-end tests for LSTM and GRU layers. For this, I implemented parse functions for LSTM and GRU layer in MethodDL.cxx file, which creates the corresponding layer by parsing its string description. I also implemented AddBasicLSTMLayer() and AddBasicGRULayer() functions in the DeepNet.h file which add these layers to a deep-net.

For the end-to-end testing I used a dataset called tmva_class_example.root which was also used for testing RNN. I implemented end-to-end tests files named LSTMClassification.C and GRUClassificatioon.C which are not included in the pull request but may be used as tutorials. In these files I created a deep-net consisting of a LSTM(or GRU) layer followed by a reshape layer and two dense layers. This deep-net was used for classifying events as signal or backgroud. The test was performed for both architecture types(cpu and ref). The cpu training time was observed to be much less than ref(almost by half).
